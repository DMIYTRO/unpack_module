# unpack_module

Автоматический пайплайн для распаковки архивов с макетами и переименования файлов по заказам с сайта Сборка UA.

## Как запустить

```bash
# 1. Создать виртуальное окружение и установить зависимости
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. Создать файл .env с доступами (скопировать из примера)
cp .env.example .env
# Заполнить SBORKA_API_KEY в .env

# 3. Положить архивы RAR или ZIP в папку original_archives/

# 4. Запустить
python3 api_core.py
```

## Структура файлов

| Файл | Назначение |
|---|---|
| `main.py` | Главный оркестратор пайплайна |
| `api_core.py` | Новое ядро: распаковка и переименование через API sborka.ua |
| `unpack.py` | Распаковка RAR-архивов |
| `validator.py` | Проверка соответствия количества файлов сторонности |
| `classifier.py` | Определение лицо/оборот по имени файла |
| `renamer.py` | Переименование по локальной логике (простые заказы) |
| `filename_parser.py` | Парсинг метаданных из имени папки/архива |
| `website_parser.py` | Клиент `action=getSubOrders` API sborka.ua |

## Логика нового ядра

1. **Поиск** — ядро находит все RAR и ZIP во входном каталоге.
2. **Распаковка** — каждый архив безопасно распаковывается в одноимённую папку.
3. **API** — номер основного заказа извлекается из имени, после чего выполняется только запрос `action=getSubOrders`; основной номер добавляется к полученным подзаказам.
4. **Проверка** — количество макетов должно точно совпасть с общим числом заказов и сторонностью (`4-4` = 2 файла, `4-0` = 1 файл).
5. **Переименование** — вся карта имён проверяется заранее и применяется атомарно.
6. **Завершение** — обработанный архив переносится в `_DONE_`, а папка получает маркер `.done`.
7. **Защита** — несовпадения уходят в `_REQUIRES_MANUAL_CHECK_`; при сетевой ошибке исходные файлы остаются на месте для повтора.

Можно указать отдельный выходной каталог:

```bash
python3 api_core.py /путь/к/архивам /путь/к/распакованным
```

## Работа с папками

Архивы кладутся непосредственно во входной каталог. Имя должно содержать сторонность (`4-0`, `4-4`) и номер заказа в блоке вида `(33342-25509667)`.

При запуске с одной папкой:

```bash
python3 api_core.py /путь/к/incoming
```

результат выглядит так:

```text
incoming/
├── имя_заказа/                 # макеты и маркер .done
├── _DONE_/                    # успешно обработанные архивы
├── _REQUIRES_MANUAL_CHECK_/   # несовпадение файлов и API
├── _TROUBLES_/                # битые, опасные или слишком большие архивы
└── rename_log.txt
```

При запуске с двумя папками архивы берутся из первой, а распакованные заказы и журнал создаются во второй. Исходные архивы после обработки перемещаются в служебные каталоги первой папки.

## Ограничения и старые кодировки

- Максимальный размер одного архива — **1,5 ГБ (1 500 000 000 байт)**. Более крупный архив без распаковки перемещается в `_TROUBLES_` с файлом причины.
- ZIP с правильным UTF-8 обрабатываются без изменения имён.
- Для старых ZIP из WinRAR и Проводника восстанавливаются имена CP866 и CP1251 без UTF-8-флага, включая вложенные каталоги.
- После восстановления имени повторно проверяется защита от выхода за рабочую папку.
- Символические ссылки и повторяющиеся пути внутри ZIP отклоняются.

Для RAR требуется `unar`, `unrar` или `7z`:

```bash
# macOS
brew install unar

# Ubuntu/Debian
sudo apt install unar
```

На Windows установите 7-Zip, добавьте `7z.exe` в `PATH`, создайте окружение командой `py -m venv .venv` и активируйте его через `.venv\Scripts\activate`.
