# unpack_module

Автоматический пайплайн для распаковки архивов с макетами и переименования файлов по заказам с сайта Сборка UA.

## Как запустить

```bash
# 1. Создать виртуальное окружение и установить зависимости
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
playwright install chromium

# 2. Создать файл .env с доступами (скопировать из примера)
cp .env.example .env
# Заполнить LOGIN_USER и ADMIN_PASSWORD в .env

# 3. Положить архивы RAR в папку original_archives/

# 4. Запустить
python3 main.py
```

## Структура файлов

| Файл | Назначение |
|---|---|
| `main.py` | Главный оркестратор пайплайна |
| `unpack.py` | Распаковка RAR-архивов |
| `validator.py` | Проверка соответствия количества файлов сторонности |
| `classifier.py` | Определение лицо/оборот по имени файла |
| `renamer.py` | Переименование по локальной логике (простые заказы) |
| `filename_parser.py` | Парсинг метаданных из имени папки/архива |
| `website_parser.py` | Playwright-парсер подзаказов с sborka.ua |

## Логика работы

1. **Распаковка** — все RAR из `original_archives/` распаковываются в подпапки.
2. **Валидация** — для каждой папки проверяется, совпадает ли количество файлов с заявленной сторонностью (`4-4` = 2 файла, `4-0` = 1 файл).
3. **Простой режим** — если всё совпадает, файлы переименовываются локально (лицо/оборот).
4. **Сложный режим** — если файлов больше, чем ожидается, скрипт заходит на сайт и получает список подзаказов.
5. **Предпросмотр** — оператор видит таблицу привязки файлов к подзаказам и подтверждает.
6. **АЛЕРТ** — если количество не совпадает с сайтом, папка уходит в `_REQUIRES_MANUAL_CHECK_/` с объяснением причины.
7. **Маркер** — успешно обработанные папки помечаются `.done`, чтобы при повторном запуске их не трогать.
